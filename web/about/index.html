About

Fullstack.wiki is a repository of documentation for how to build reliable, accessible, cross-platform Web applications.

Authoring

Content is authored in bare HTML files annotated with RDFa. Data about the content is embedded using RDFa, extracted from the documents, and used to create lists of content elsewhere.

Nits

Often while looking in retrospect, we wouldn't have designed standards the way they were. We should learn from these mistakes and point out the right way to do it, so they don't get repeated. These should be pointed out as soon as they're introduced, if possible. Articles should have a section called "Nits" that describes the problems, and the right way to go about them.

These problems might include things such as:

  • Misspellings
  • Inconstistencies between standards
  • Willful violations of standards
  • Historical reasons for bad behavior
